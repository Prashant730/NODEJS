How This Works:

  1. You enter username and password
  2. Click "Submit"
  3. Browser sends POST request with data in body (not URL)
  4. Server reads data from req.body (requires body-parser)
  5. Server processes and returns JSON response

POST vs GET:

Feature GET POST
Data Location URL (visible) Request body (hidden)
Security Less secure More secure
Data Size Limited (~2KB) Unlimited
Bookmarkable Yes No
Use Case Search, filters Login, registration

Note: Watch the URL bar after submitting - it stays the same! Data is sent securely in the request body.
